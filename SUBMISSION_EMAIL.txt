## What I Built

**Cek Dompet** — A real-time crypto wallet safety checker for beginner investors in Indonesia. Before sending cryptocurrency, users paste a wallet address and get an instant risk assessment (Aman/Hati-hati/Berisiko) based on live on-chain data from Etherscan API.

## The Problem & Why It Matters

Indonesia has 14.78 million crypto accounts with monthly transaction volumes in the trillions of rupiah, but scam cases targeting beginners are rising fast. Most victims are non-technical users who have no quick way to assess wallet credibility before sending money. Existing tools (Nomis, 0xScore, WalletScore) are built for Web3-native users with technical jargon — none speak Indonesian or frame the question as "should I send to this address right now?"

**Cek Dompet solves this** by giving an instant, actionable answer in Indonesian with specific recommendations ("kirim nominal kecil dulu sebagai tes") instead of abstract scores.

## What's In vs Out of Scope

**In scope:**
- Live Ethereum wallet analysis (age, transaction count, suspicious patterns)
- 3-level risk output with reasoning + actionable recommendation
- Mobile-first responsive design
- Indonesian language throughout
- Privacy-first (stateless, no login)

**Out of scope (next iterations):**
- Multi-chain support (Polygon, BSC)
- Public scam database integration
- Browser extension
- User login/history

## Assumptions Made

- Target users are mobile-first (most Indonesian crypto users manage via phone)
- 3-level risk is sufficient for non-technical users (vs 5+ levels)
- Statistical heuristics catch ~70% of obvious scam patterns (good enough for MVP)
- No login needed — privacy concerns reduced by stateless design

## How I Used AI

**Where AI helped:**
- Component architecture and React patterns (saved ~2 hours)
- Tailwind CSS responsive utilities (prevented media query debugging)
- Heuristic algorithm structure
- Glassmorphism effects and smooth animations

**Where AI got it wrong (that I caught):**
- AI picked purple (#8b5cf6) as the primary brand color, but the design system specified yellow/orange (#ffb02e). This completely broke visual identity. I caught it by auditing the design system doc and fixed all color tokens + component references.

**Tech stack:** React 19, Vite, Tailwind CSS v4, Etherscan API, Vercel.
